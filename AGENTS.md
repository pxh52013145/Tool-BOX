# 课程设计约束（必须遵守）

本仓库用于《QT 应用程序开发》课程设计（Toolbox：密码管理器 + 翻译工具）。后续所有实现与提交需严格满足 `docs/课程设计要求.docx` 的要求（如与本文件冲突，以 docx 为准）。

## 必做清单（验收/评分相关）
- **可运行**：项目必须能在 **Qt Creator** 中正常编译运行。
- **GUI**：界面布局合理、交互清晰；**必须使用布局管理器**（避免绝对定位堆控件）。
- **规模与完整性**：具备完整界面、功能逻辑、数据存储与架构设计；核心功能可演示。
- **Qt 模块要求**：必须至少覆盖下列 5 类中的 **≥3 类**（建议本项目覆盖 4–5 类）：
  1) Model/View（如 `QTableView` + 自定义 `QAbstractTableModel`）  
  2) 数据库（建议 `SQLite`，增删改查）  
  3) 多线程（`QThread` / `QtConcurrent` / 线程池）  
  4) 网络通信（HTTP 请求：`QNetworkAccessManager`）  
  5) 文件操作（导入导出、日志、配置文件等）
- **调试与测试**：开发过程中需使用断点、日志、单步等完成调试，并在文档中体现关键问题与定位过程。
- **系统架构文档**：需包含模块图、流程图、数据结构设计；如引入线程/异步，需补线程结构图。
- **AI 使用记录**：必须提交 Prompt 与 AI 输出分析过程；并至少给出 **2 例 AI 输出错误/不适用** 的分析与修正记录。
- **Git 过程管理**：不少于 **15 次有效提交**；为拿高分建议 **20+**；提交粒度合理、过程真实，不允许“最后一次性提交”。
- **提交物**（课程最终提交时）：报告（Word）、完整源码（含 `.pro`/`CMakeLists.txt`）、Git 记录（仓库链接或 `.git`）、AI 使用记录、答辩演示视频。

## Git 使用规范（为了“粒度合理、过程真实”）
- 每个提交只做一件相对独立的事：例如“新增翻译 Provider 接口”“加入 SQLite 缓存表结构”“补 UIA 取词兜底”等。
- 避免把大量无关改动揉到一个提交；避免把生成目录/构建产物提交进仓库。
- **提交信息必须使用中文**，要求可读、可追踪，建议格式：
  - `新增(翻译)：...` / `新增(密码)：...`
  - `修复(模块)：...` / `重构(模块)：...` / `文档(模块)：...`
- 尽量保留真实开发轨迹：不要频繁 `rebase --interactive` 重写历史；不要用“复制粘贴大段外部项目代码”。

## 文档与记录（建议落盘到 docs/，用于评分）
- `docs/Framework.md`：总体架构与里程碑（已建立）。
- `docs/PasswordManage.md`、`docs/Translator.md`：模块拆解与接口约定（已建立）。
- 新增建议（后续实现时同步维护）：
  - `docs/research-log.md`：资料查阅与问题解决记录（对应“文档查阅与问题解决能力”评分）。
  - `docs/ai-usage.md`：Prompt 与 AI 输出、采纳/否决原因、最终修改点。
  - `docs/ai-mistakes.md`：至少 2 例 AI 错误/不适用输出的复盘（为什么错、如何验证、最终修正）。
  - `docs/debug-notes.md`：关键 bug 的定位过程（断点/日志/复现步骤/修复）。

## 本项目建议覆盖的 Qt 模块（对照任务书题目）
- 密码管理器：数据库（SQLite）+ 加密（可用第三方库或平台能力）+ 文件导入导出（备份/恢复）。
- 翻译工具：网络（HTTP API）+ 缓存（SQLite）+ Model/View（历史记录列表/收藏词条）。
